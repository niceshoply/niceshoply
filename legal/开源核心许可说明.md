# 开源核心许可说明（MIT）

## 核心承诺：你拥有完整源码

NiceShoply 采用 **MIT License** 发布主程序源码。这意味着：

1. **完整源码公开**：GitHub 仓库包含全部 PHP、前端、配置与迁移文件，无隐藏闭源模块。
2. **可自托管**：部署在你自己的服务器/VPS/容器，数据与代码完全在你掌控之下。
3. **可任意修改**：改主题、改结账流程、接支付、写插件——无需官方审批。
4. **可商用**：用自己的店铺卖货、给客户建站、做 SaaS 封装（须遵守商标政策）。
5. **无 GMV 抽成**：自托管使用核心功能不向 NiceShoply 缴纳交易佣金。

这与 **Shopify**（及多数 SaaS 电商）形成鲜明对比：Shopify 不提供源码，店铺与数据实质托管于平台，深度定制与迁移均受限于平台规则。

## 常见问题

### Q：我用 NiceShoply 开店，需要付钱吗？

**不需要。** 自托管 + 核心功能永久免费。只有当你需要官方高级插件、托管云服务，或商业白标时才会产生费用。

### Q：平台会锁我功能吗？

**不会。** 核心不采用「功能阉割式」收费。你的源码在本地，即使官方服务停运，你已部署的实例仍可继续运行（需自行维护安全更新）。

### Q：和 Shopify 比，源码所有权意味着什么？

| Shopify | NiceShoply |
|---------|------------|
| 无法查看/修改底层代码 | 完整 Laravel 源码可审计、可改 |
| 依赖平台 API 与主题体系 | 直接改 PHP、Blade、Vue |
| 迁移=重建 | 数据库+代码打包即可迁移 |
| 平台规则变更直接影响店铺 | 规则变更你可选择不升级 |

### Q：从 OSL-3.0 变更为 MIT 会影响我吗？

MIT 比原 OSL-3.0 **更宽松**：移除了「须保留官方应用市场」等附加限制，更符合「最大限度开源」定位。你获得更大的定制自由。
